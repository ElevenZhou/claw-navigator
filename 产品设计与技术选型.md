## 四、技术选型

### 方案对比

| 维度 | 方案A：纯静态 | 方案B：Astro | 方案C：Next.js |
|------|-------------|-------------|---------------|
| **技术** | HTML + Tailwind + JS | Astro + Tailwind | Next.js + Tailwind |
| **数据源** | JSON 文件 | JSON/MDX | 数据库(API) |
| **SEO** | ⭐⭐ 需手动优化 | ⭐⭐⭐⭐⭐ 原生SSG | ⭐⭐⭐⭐⭐ SSR/SSG |
| **交互** | ⭐⭐ 原生JS | ⭐⭐⭐⭐ 岛屿架构 | ⭐⭐⭐⭐⭐ React全量 |
| **部署** | 任意静态托管 | Vercel/Netlify | Vercel/自有服务器 |
| **开发速度** | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| **维护成本** | ⭐⭐ 低 | ⭐⭐⭐ 中 | ⭐⭐⭐⭐ 高 |
| **适合场景** | MVP快速验证 | 内容站最佳 | 复杂交互应用 |

### ✅ 推荐方案：**Astro + Tailwind CSS**

**理由**：

1. **内容站首选** — Astro 天生为内容驱动型网站设计，Claw 导航站本质就是内容站
2. **SEO 极佳** — 默认生成静态 HTML，零 JS 也能渲染，搜索引擎完美收录
3. **性能极好** — "岛屿架构"：页面默认零 JS，只有交互组件按需加载，首屏极快
4. **开发效率高** — 组件可用 React/Vue/Svelte，但最终输出纯静态
5. **部署简单** — 生成静态文件，丢到任意静态托管即可（GitHub Pages、Vercel、甚至 Nginx）
6. **数据管理** — 用 JSON/MDX 管理产品数据，新增产品只需加一条 JSON 记录

### 技术栈详情

```
前端框架：Astro 5.x
样式方案：Tailwind CSS 4.x
UI 组件：React Islands（交互部分）
数据存储：JSON 数据文件（产品信息）
图标：Lucide Icons / Heroicons
字体：Noto Sans SC（中文）+ Inter（英文）
部署：Vercel（推荐）或 Netlify
域名：clawnav.com / clawmap.cn / longxia.io（待定）
包管理：pnpm
```

## 五、设计风格

### 视觉风格
- **主题**：深色科技感 + 龙虾橙（#FF6B35）作为强调色
- **布局**：卡片式，大量留白，信息密度适中
- **响应式**：移动端优先，桌面端三列、平板两列、手机一列
- **动效**：卡片悬浮提升、筛选切换过渡、对比表滑入

### 配色方案
```
背景色：#0A0A0F（深黑蓝）
卡片色：#16161F（深灰蓝）
文字色：#E8E8ED（浅白）
强调色：#FF6B35（龙虾橙）
次要色：#4ECDC4（青绿）
标签色：按阵营分配（腾讯蓝/阿里橙/百度红/字节绿...）
```

## 八、风险与对策

| 风险 | 对策 |
|------|------|
| 产品更新快，信息过时 | 数据与页面分离，JSON 更新即可，无需改代码 |
